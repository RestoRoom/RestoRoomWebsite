🔍 How it works

The plan.

Reimplementing a game server is a big job. Here's our approach, broken down step by step.

1

Capture traffic

Using tools like Wireshark and mitmproxy, we intercept and log all communication between the Rec Room client and official servers — REST API calls, Photon packets, asset requests, everything. This needs to happen before June 1st.

2

Reverse engineer the client

Rec Room is a Unity/C# game. Using tools like dnSpy, we read the client code to understand exactly how it communicates — filling in gaps that traffic capture alone can't show us.

3

Reimplement the servers

With a full picture of the protocol, we write our own server that speaks the same language as the official one — open source, community run, and free forever.

4

Redirect the client

We patch or proxy the official client so it connects to RestoRoom servers instead of the now-dead official ones. No brand new client required — just the one you already have.
